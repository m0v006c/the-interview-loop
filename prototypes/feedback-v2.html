Reference solution · Number of Islands
How an experienced candidate would walk through this
Short version: grid traversal — for every unvisited land cell, run BFS/DFS that marks all connected land as visited, and increment a counter. O(m × n) time and space.

1 · Clarify

  • Is adjacency 4-directional or 8-directional? (usually 4)
  • Can I mutate the input grid? (yes, saves O(m×n) visited memory)
  • What's the max grid size? (affects recursion-vs-iterative decision)

2 · Approach

Iterate every cell. When you see a '1' that hasn't been visited, that's a new island — increment the counter, then flood-fill all connected land so you don't count it again.

Flood fill can be BFS (queue), DFS recursive, or DFS iterative (stack). Iterative is safest for large grids to avoid stack overflow.

3 · Code (Python, BFS)

from collections import deque

def numIslands(grid):
    if not grid or not grid[0]:
        return 0
    rows, cols = len(grid), len(grid[0])
    count = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                count += 1
                # BFS flood-fill, mutating in-place
                q = deque([(r, c)])
                grid[r][c] = '0'
                while q:
                    i, j = q.popleft()
                    for di, dj in ((-1,0),(1,0),(0,-1),(0,1)):
                        ni, nj = i + di, j + dj
                        if 0 <= ni < rows and 0 <= nj < cols and grid[ni][nj] == '1':
                            grid[ni][nj] = '0'
                            q.append((ni, nj))
    return count

4 · Complexity

  • Time: O(m × n). Every cell is visited once; each visit does O(1) work.
  • Space: O(m × n) worst-case for the BFS queue (e.g., one big island).

5 · Common pitfalls

  • Stating O(N log N) — no, there's no sorting; it's pure traversal.
  • Forgetting to mark visited → infinite loop.
  • Using DFS recursion on huge grids → stack overflow (use iterative or BFS).
  • Counting '1' as an integer instead of a string — the input is character grid.

6 · Alternative: Union-Find

Model each land cell as a node; union adjacent land cells. Final island count = number of roots with land. Near-linear with path compression + union by rank. Shines if you need to answer connectivity queries over time; for a one-shot count, BFS/DFS is simpler.
